##必败态（P）##：必败态就是指前一玩家（pre-player）将取胜的状态

##必胜态（N）##：必胜态就是指后一玩家（next-player）将取胜的状态

关于他们的属性有：
1.所有的终点都是必败态
2.从必胜态操作，一定有一个方法可以进入必败态。
3.从必败态操作，只能进入必胜态。

关于nim有一个基本的定理那就是，当你的堆为（x1,x2…xn）时他们的异或和是0，则当前点为必败态。

知道了上面的那个基本定理的话，最基本的nim是可以做了，先来练手。。

hdu1850

就是最基本的nim博弈，要求输出可行的方案总数。于是我们先求出所有数的异或和，先判断是不是0，如果是0，当前就已经是必败态了。如果不是就遍历数组，取出一个数来后判断这个数是不是大于其余数的异或和，如果大于，那么久可以从那个状态转化到一个必败态。

#include <bits/stdc++.h>
using namespace std;

int main()
{
    int n;
    while(cin >> n && n)
    {
        int a[111],t = 0;
        for(int i = 0;i < n;i++)
        {
            cin >> a[i];
            t ^= a[i];
        }
        if(t == 0)
            cout << 0 << endl;
        else
        {
            int sum = 0;
            for(int i = 0;i < n;i++)
            {
                t ^= a[i];
                if(a[i] > t)
                    sum++;
                t ^= a[i];
            }
            cout << sum << endl;
        }
    }
}

然后引入SG定理，sprague-grundy定理

我们可以把游戏转化成一个有向无环图，每个点对应一个状态，每条边代表一个合法的操作就可以把所有的点标出状态。

sg函数的求解的话，就在于要了解一个mex运算，这是一个施加在整数集合上的运算，表示最小的不属于这个集合的非负整数，比如mex{1,2,4} = 3。

例如：取石子问题，有1堆n个的石子，每次只能取{1，3,4}个石子，先取完石子者胜利，那么各个数的SG值为多少？

sg[0]=0，f[]={1,3,4},

x=1时，可以取走1-f{1}个石子，剩余{0}个，mex{sg[0]}={0}，故sg[1]=1;

x=2时，可以取走2-f{1}个石子，剩余{1}个，mex{sg[1]}={1}，故sg[2]=0；

x=3时，可以取走3-f{1,3}个石子，剩余{2,0}个，mex{sg[2],sg[0]}={0,0}，故sg[3]=1;

x=4时，可以取走4-f{1,3,4}个石子，剩余{3,1,0}个，mex{sg[3],sg[1],sg[0]}={1,1,0},故sg[4]=2;

x=5时，可以取走5-f{1,3,4}个石子，剩余{4,2,1}个，mex{sg[4],sg[2],sg[1]}={2,0,1},故sg[5]=3；

以此类推…..

x 0 1 2 3 4 5 6 7 8….

sg[x] 0 1 0 1 2 3 2 0 1….

然后扔一个求SG函数的模板

//f[]：可以取走的石子个数，即规则
//sg[]:0~n的SG函数值
//has[]:mex{}
int f[N],sg[N],has[N];   
void getSG(int n)
{
    int i,j;
    memset(sg,0,sizeof(sg));
    for(i=1;i<=n;i++)
    {
        memset(has,0,sizeof(has));
        for(j=1;f[j]<=i;j++)
            has[sg[i-f[j]]]=1;
        for(j=0;j<=n;j++)    //求mes{}中未出现的最小的非负整数
        {
            if(has[j]==0)
            {
                sg[i]=j;
                break;
            }
        }
    }
}

关于SG值的用处：SG值为零的点，则为P点(即，必败点)，否则为N点(即，必胜点)。以上SG值的性质满足必败点与必胜点的所有性质:(1)所有终结点的SG值为零，为必败点。(2)若SG值不为零，则其为必胜点。设其值为m，则有SG=g(x)={0,1,2,….m-1}=m，大括号中的0为其后继点的SG值，即该点为必败点。所以满足从必胜点至少有一种方法进入必败点。(3)若，SG值为零的点不是终结点，则必存在一个值为m的点，使得SG=g(x)={…，m，…}，大括号中的m为其后继点的SG值，即该点为必胜点。所以满足从必败点都只能进入必胜点。

#include <bits/stdc++.h>
using namespace std;

int f[10001];  
unsigned s[101];  
int sNum;  

int mex(int v)  
{  
    bool g[101]={0};  
    for (int i=0;i<sNum;i++)  
    {  
        int t=v-s[i];  
        if(t<0)  
            break;  
        if(f[t]==-1)  
            f[t]=mex(t);  
        g[f[t]]=1;  
    }  
    for (int i=0;i<101;i++)  
    {  
        if(!g[i])  
            return i;  
    }  
}

int sg[100005];

// N求解范围 S[]数组是可以每次取的值，t是s的长度。
void sg_solve(int *s, int t, int N) {
    int i,j;
    bool hash[N];
    memset(sg,0,sizeof(sg));
    for(i=1;i<=N;i++) {
        memset(hash,0,sizeof(hash));
        for(j=0;j<t;j++) {
            if(i - s[j] >= 0) {
                hash[sg[i-s[j]]] = 1;
            }
        }
        for(j=0;j<=N;j++) {
            if(!hash[j]) { break; }
        }
        sg[i] = j;
    }
}
